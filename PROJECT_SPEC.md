# Voice Agent Platform - Architecture & Project Specification

## 1. Overview
The Voice Agent Platform is a multi-tenant backend solution that allows "Resellers" to create and manage AI Voice Agents for their "Customers." It bridges **Supabase** (Database & Auth), **Retell AI** (Voice LLM & Telephony), and **OpenRouter** (for routing LLM requests, if needed) into a single unified FastAPI backend.

---

## 2. Tech Stack
*   **Backend Framework:** FastAPI (Python 3.12)
*   **Server:** Uvicorn
*   **Database & Authentication:** Supabase (PostgreSQL + JWT Auth)
*   **Voice AI Engine:** Retell AI (creates LLMs, handles Twilio/Phone calls, STT/TTS)
*   **Deployment:** Railway / Render (Docker/Nixpacks automated deployments)

---

## 3. Database Schema (Supabase)

### `resellers`
The top-level entity. Users who log into the dashboard.
*   `id` (uuid, PK) - Maps to Supabase Auth UUID.
*   `name` (text)
*   `email` (text)

### `customers`
Created by resellers. Customers "own" the voice agents.
*   `id` (uuid, PK)
*   `reseller_id` (uuid, FK)
*   `name` (text)
*   `billing_email` (text)
*   `phone_number` (text, nullable)
*   `plan` (text) - e.g. "starter"
*   `status` (text) - e.g. "active"
*   `retell_agent_id` (text) - The ID generated by Retell AI.

### `agent_configs`
The specific prompt and voice setting assigned to a customer's agent.
*   `id` (uuid, PK)
*   `customer_id` (uuid, FK, Unique)
*   `system_prompt` (text)
*   `voice_id` (text) - e.g. `11labs-Adrian`
*   `language` (text) - e.g. `en-US`
*   `llm_model` (text)
*   `recording_enabled` (bool)
*   `business_hours`, `escalation_phone`, `calendar_webhook_url`, `crm_webhook_url`, `faq_knowledge_base`

### `calls`
Logs of telephone calls processed by Retell AI webhooks.
*   `id` (uuid, PK)
*   `customer_id` (uuid, FK)
*   `retell_call_id` (text)
*   `caller_number` (text)
*   `duration_seconds` (int)
*   `outcome` (text) - e.g. "completed", "transferred", "voicemail"
*   `transcript` (text)

## 4. Environment Variables (`.env`)
Required to run the application securely:
*   `DEV_MODE` (true/false) - Bypasses Retell API calls if true, allowing local frontend/mock testing without a credit card.
*   `SUPABASE_URL` - Supabase project URL.
*   `SUPABASE_SERVICE_KEY` - Supabase service role key (bypasses RLS to act on behalf of the backend).
*   `SUPABASE_JWT_SECRET` - The signing key to verify Reseller Login tokens.
*   `RETELL_API_KEY` - API key for Retell AI.
*   `OPENROUTER_API_KEY` - Optional for OpenRouter fallback.

## 5. API Endpoints

All non-webhook endpoints require an `Authorization: Bearer <token>` header containing a valid Supabase JWT linked to a reseller account.

### A. Create Customer & Voice Agent
**`POST /api/customers`**
Creates a Retell LLM, a Retell Agent, saves the Customer to DB, and inserts the Agent Config to the DB.

**Request Body:**
```json
{
  "name": "Jane Doe",
  "billing_email": "jane@example.com",
  "phone_number": "+15551234567",
  "plan": "starter",
  "status": "active",
  "retell_agent_id": null, // (Optional) Pass an existing ID to skip Retell creation
  "agent_config": {
    "system_prompt": "You are a customer service assistant...",
    "voice_id": "11labs-Adrian",
    "language": "en-US",
    "llm_model": "gpt-4o",
    "recording_enabled": true
  }
}
```

**Response (201 Created):**
Returns the newly created Customer row object from Supabase including the generated UUID and Supabase `created_at` timestamp.

---

### B. List Customers
**`GET /api/customers`**
Fetches all customers that belong exclusively to the logged-in Reseller.

**Response (200 OK):**
```json
[
  {
    "id": "customer-uuid-here",
    "reseller_id": "reseller-uuid-here",
    "name": "Jane Doe",
    "retell_agent_id": "agent_abc123",
    ...
  }
]
```

---

### C. Update Agent Configuration
**`PUT /api/customers/{customer_id}/config`**
Patches the `agent_configs` table in Supabase AND performs a live `PATCH` update to the LLM/Agent settings on Retell AI's servers.

**Request Body:**
```json
{
  "agent_config": {
    "system_prompt": "You are now a hyper-focused sales AI...",
    "language": "en-US",
    "recording_enabled": false
  }
}
```

**Response (200 OK):**
Returns the updated `agent_configs` row from Supabase.

---

### D. List Call Logs
**`GET /api/calls`**
Fetches call logs from the database, performing an inner join against `customers` to guarantee the logged-in reseller is only viewing calls belonging to their own clients.

**Response (200 OK):**
```json
[
  {
    "id": "uuid",
    "customer_id": "customer-uuid",
    "retell_call_id": "call_xyz",
    "caller_number": "+19998887777",
    "duration_seconds": 120,
    "transcript": "Hello, how can I help? ...",
    ...
  }
]
```

---

### E. Retell AI Webhook Receiver
**`POST /api/webhook/retell`**
Listens for POST events fired automatically by Retell AI when a phone call starts, is analyzed, or ends. No auth required (protected by Retell Signature validation in production).

**Workflow Actions:**
1. Validates event (`call_ended` or `call_analyzed`).
2. Extracts the `transcript`, `duration_ms`, `from_number`, and `disconnection_reason`.
3. Fetches the associated `customer_id` by querying Supabase `customers` table using the Retell `agent_id`.
4. Saves/Upserts the call data into the Supabase `calls` table asynchronously.
